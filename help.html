    Technical reference for the IFA administrator. This section is in English only.

    Architecture Overview

    There are two submission paths depending on how index.html is opened:

    ── Primary path (index.html embedded in fistball.org iframe) ────────

Evaluator fills index.html
        │
        ▼
  Tap ✉ Submit
        │
        ▼
  window.postMessage → parent fistball.org page receives payload
  (parent handles storage / further processing)

── Fallback path (index.html opened standalone) ─────────────────────

Evaluator fills index.html
        │
        ▼
  Tap ✉ Submit
        │
        ▼
  mailto: link opens email client
  (body contains JSON payload)
        │
        ▼
  Power Automate flow (email trigger)
  ├── Parse JSON from email body
  └── Create item in SharePoint List

    Step 2 — Power Automate Flow

    ⚠️ This flow is only triggered when index.html is opened standalone. When embedded in the fistball.org iframe, submission uses postMessage and bypasses this flow entirely.

    Trigger: When a new email arrives (V3) — Office 365 Outlook, filtered to the evaluation mailbox.

    Step 1 — Compose: Extract JSON

    substring(
  body('Get_email')?['body'],
  indexOf(body('Get_email')?['body'], '{&quot;'),
  sub(
    lastIndexOf(body('Get_email')?['body'], '&quot;}'),
    indexOf(body('Get_email')?['body'], '{&quot;')
  )
)

    Uses {&quot; (not {) to skip CSS curly braces in the HTML-encoded email body.

    Step 2 — Compose: Decode HTML entities — chain replace() calls to convert &quot; → ", &amp; → &, &lt; → <, &gt; → >, &#39; → '.

    Step 3 — Parse JSON — use the decoded output. Click Generate from sample with a real exported evaluation JSON to build the schema. Re-generate if new fields are added.

    Step 4 — Create item in SharePoint — site: referee, list: RefereeEvaluations. Map all Parse JSON output fields to the corresponding SharePoint columns.

    Step 3 — SharePoint File Hosting

    Upload index.html and help.html to Site contents → SiteAssets. Navigate into the file and copy the URL from the address bar (not the sharing link) to get the direct executable URL.

    If the page loads but nothing happens, the site may have NoScript enabled. Fix: Set-SPOSite -Identity … -DenyAddAndCustomizePages $false

    Step 4 — Updating Files

    Upload the new version to SiteAssets — SharePoint replaces the existing file. Users may need to clear cache (Ctrl+Shift+R). No changes to the list or flow are needed for UI updates. If the email payload structure changes, regenerate the Parse JSON schema from a sample.

    Troubleshooting

    SymptomCause & Fix
    Power Automate: "Invalid JavaScript property identifier character: -"Compose formula found a CSS { before the JSON. Search for {&quot; instead of {.
    Power Automate: "Schema validation failed"Payload has new fields. Regenerate schema via Generate from sample.

    Maintenance Notes

    • List permissions: Inherited from the SharePoint site. Adjust if tighter access control is needed.
    • Power Automate licence: Uses standard connectors only (Office 365 Outlook, SharePoint). A standard M365 Business licence is sufficient.
